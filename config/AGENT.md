# 通用协作规则
- 具体执行红线统一维护在同级目录 `force/collaboration-rule.md`
- 代码开发、文档编写等相关任务时，必须读取并遵守 `force/collaboration-rule.md`。
- `force/collaboration-rule.md` 为最低优先级兜底规则；若本文件、项目规则或用户明确要求与其冲突，优先遵守更具体的约束。

# 技能位置
- 当用户提到 `Codex 技能`、`Codex 全局 skills`、`Codex skills 目录` 时，默认对应本地目录 `~/.codex/skills/`。
- 当用户提到 `Claude 技能`、`Claude 全局 skills`、`Claude skills 目录` 时，默认对应本地目录 `~/.claude/skills/`。
- 处理这两类请求时，优先直接使用上述固定目录，不再重复做全盘查找；除非用户明确说明使用其他目录。

# 文档规范
- 写技能、规范、校验脚本或自动化约束时，只沉淀最新事实和稳定规则，优先抽象出通用不变量。
- 不要把当前案例、具体文档类型、章节编号、标题名称、工具运行目录或临时实现写死，也不要留下迁移说明、来源解释、自我注释或“某内容已放到某处所以这里不写”的流水账。
- 只有稳定对外接口、机器读取契约或用户明确要求固定的字段，才应做结构化强约束。
